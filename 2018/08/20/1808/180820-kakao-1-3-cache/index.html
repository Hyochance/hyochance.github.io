카카오 1차 예선 - 3번 문제 Cache

2018-08-20
KAKAO, 알고리즘

코멘트

문제는 http://tech.kakao.com/2017/09/27/kakao-blind-recruitment-round-1/ 에서 확인할 수 있다. 이 문제를 푸는데 총 소요한 시간은 2시간 5분… 삽질한 시간을 줄일 수만 있다면 얼마나 좋을까!

처음에는 문제를 읽고도 요구하는게 뭔지 잘 몰랐다. LRU 캐시 교체 알고리즘? 캐시를 구현하라고? 이게 난이도 : 하 라고??등등… 혼란스러워 하다가 우선 할 수 있는데까지 해보기로 했다.

그래서 입력 부분을 구현하고, “Jeju”, “Pangyo”와 같은 문자열을 jeju, pangyo로 치환하는 문자열 처리까지는 완료했다. 마지막으로 가장 중요한 캐시 부분을 구현해야 하는데 도저히 아이디어가 떠오르지 않았다. 그 때! 1차 예선은 인터넷 검색도 가능한 오픈북 시험이란 것을 떠올렸다. 바로 구글로 가서 LRU캐시에 대해 검색해보고, 원리를 이해하고, java로 구현되어 있는 꽤 깔끔한 코드를 찾아서 바로 적용해봤다.

그런데 문제가 생겼다. 이 코드는 키와 값을 기반으로 하는 map구조의 LRU캐시였다. 내가 필요한건 단순히 도시의 이름만을 비교하면 되는 간단한 LRU캐시인데… 그래서 열심히 코드를 수정해봤다. 그런데 건드릴수록 프로그램은 곳곳에서 타입에러를 뿜어대며, 나도 내가 뭘 고치고 있는지 모르는 지경에 이르렀다.

그래서 멈췄다. 여기까지 소요시간 1시간 35분. 실제 시험이였다면 이미 탈락이였겠지. 컴퓨터 앞을 벗어나 침대에 벌러덩 누워 생각했다. LinkedList가 핵심이라면, 코드를 복붙해오는것 보다 내가 직접 구현할 수도 있을 것 같은데?

옳타거니. 다시 침대에서 일어나 컴퓨터 앞으로 왔다. LinkedList는 java에서 기본 자료구조로 제공하므로, 내가 일일이 Node나 별에별 메소드를 구현하지 않아도 된다. 게다가 java의 LinkedList는 addFirst, addLast, removeFirst, removeLast같이 필수적인 메소드를 다 제공하니 얼마나 고마운가.

그렇게 구현한것이 LRUCache 클래스다. 이 문제를 푸는데는 set 메서드 하나만 직접 구현해내면 됐다. (java의 LinkedList에서 제공하는 set과는 다르다) 이후는 일사천리였다. LRUCache에서 return해주는 “hit” 또는 “miss”에 따라 소요시간을 더해주면 된다. 단 cacheSize가 0일 때는, 말그대로 캐시가 없으므로 모든 입력은 miss가 나서, 그저 도시수 * 5해주면 소요시간이 나온다.

저번주에 카카오 1차예선 2번 문제까지 풀어놓고, 이 문제에 막혀서 일주일째 다른 문제들을 풀며 방황하다가 드디어 오늘 풀어냈다. 알고리즘 문제풀이 정말 재밌다~~(저번주에는 때려치고 싶었다.)
